# نظام ويب ذكي لدعم التشخيص الأولي للأمراض الجلدية

تطبيق Flask متكامل يربط بين واجهة الطبيب وواجهة المريض ونموذج ذكاء اصطناعي
(ResNet50 عبر تقنية نقل التعلم Transfer Learning) لتحليل صور الحالات
الجلدية وتخزين النتائج في قاعدة بيانات علائقية.

## هيكل المشروع

```
skin_disease_app/
├── app.py                 # نقطة تشغيل التطبيق وكل المسارات (Routes)
├── config.py               # إعدادات التطبيق وقاعدة البيانات
├── extensions.py           # تهيئة SQLAlchemy و Flask-Login
├── models.py                # جداول قاعدة البيانات (المستخدمين، المرضى، الحركات، التنبؤات)
├── decorators.py            # ديكوريتور التحقق من صلاحية (طبيب/مريض)
├── ml_model.py               # بناء وتحميل نموذج ResNet50 + دالة التنبؤ
├── train_model.py            # سكربت تدريب/ضبط النموذج على مجموعة بيانات جلدية
├── requirements.txt
├── static/
│   ├── css/style.css
│   └── uploads/             # الصور المرفوعة من المرضى
├── model/
│   └── resnet50_skin_disease.h5   # (ضع هنا أوزان النموذج المدرّب)
└── templates/                # صفحات HTML (Jinja2)
```

## 1) التثبيت

```bash
python -m venv venv
source venv/bin/activate      # على ويندوز: venv\Scripts\activate
pip install -r requirements.txt
```

## 2) تدريب نموذج ResNet50 (مطلوب قبل الاستخدام الفعلي)

النظام يستخدم نقل التعلم (Transfer Learning) على معمارية ResNet50. الكود
جاهز في `ml_model.py`، لكن **يجب تدريب الطبقة الأخيرة على مجموعة بيانات
جلدية حقيقية** (مثل HAM10000 من Kaggle) قبل الاعتماد عليه في تشخيص فعلي.

```bash
python train_model.py --data_dir dataset --epochs 15
```

سينتج ملف الأوزان في `model/resnet50_skin_disease.h5` وسيتم تحميله
تلقائياً عند تشغيل التطبيق. **ملاحظة مهمة:** أسماء الفئات (`CLASS_NAMES`
في `config.py`) يجب أن تطابق ترتيب المجلدات في بيانات التدريب.

بدون هذا الملف، سيعمل النظام لأغراض العرض التوضيحي فقط بأوزان ImageNet
الأصلية (نتائج التصنيف لن تكون دقيقة طبياً).

## 3) تشغيل التطبيق

```bash
python app.py
```

ثم افتح المتصفح على: `http://127.0.0.1:5000`

عند أول تشغيل يتم إنشاء قاعدة بيانات SQLite تلقائياً (`skin_disease.db`).
للانتقال إلى MySQL أو PostgreSQL، غيّر `DATABASE_URL` في `config.py` أو
كمتغير بيئة.

## 4) سيناريو الاستخدام

1. **تسجيل حساب** كمريض أو كطبيب من صفحة `/register`.
2. **المريض:** يدخل إلى "تشخيص حالة جديدة"، يرفع صورة الإصابة الجلدية،
   يضيف ملاحظات اختيارية (مثل الحكة أو مدة الإصابة)، ثم يضغط "تحليل
   الصورة". يستدعي الخادم نموذج `ResNet50` ويعرض التشخيص الأولي مع نسبة
   الثقة لكل الأمراض المحتملة، ويُخزَّن كل شيء في جدول `predictions`.
3. **الطبيب:** يدخل إلى لوحة التحكم فيرى إحصائيات عامة وأحدث الحالات،
   يمكنه فتح ملف أي مريض، مراجعة صورته وتشخيص النموذج، وإضافة ملاحظاته
   الطبية الخاصة، بالإضافة لمراجعة سجل حركات المستخدمين (Audit Log).

## 5) الأمان والخصوصية

- كلمات المرور مشفّرة عبر `werkzeug.security` (لا تُخزَّن كنص صريح).
- كل الصفحات الحساسة محمية بـ `@login_required` وديكوريتور `role_required`.
- كل عملية (تسجيل دخول، تشخيص حالة، مراجعة طبيب) تُسجَّل في جدول
  `user_actions` لأغراض التدقيق.

## ⚠️ إخلاء مسؤولية

هذا مشروع تخرج أكاديمي، والتشخيص الناتج **أولي واسترشادي فقط ولا يغني
عن استشارة طبيب مختص**، كما هو موضّح في واجهة النتائج داخل التطبيق.
